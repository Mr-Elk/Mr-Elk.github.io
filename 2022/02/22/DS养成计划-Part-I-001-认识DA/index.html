重新认识数据分析

  • 分析方法

    • 解决问题 [ <深入浅出数据分析> ]
    • 广泛涉猎 [ 36氪, 人人都是产品经理, 简书 (每日一篇, 总结与做笔记) ]
    • 产出报告 [<精益数据分析>]
  • Excel
    • 戴师兄课程
    • 刘老师的书
  • BI工具 [解决数据量较大的问题]
    • PowerBI [孙兴华zz课程]

  • Python

数据分析是什么

数据看板? 算法模型? 分析方法? [包含这些, 但是只是片面部分]

  • 观测 + 实验 + 应用
    • 观测: 对事物形成客观量化的认知 [获取数据并制作报表仪表盘]
      • 采集数据, 储存数据, 展示数据
      • 设定标准, 发现异常, 研究关系
    • 实验: 发现规律, 验证假设 [ A/B 测试]
      • 拆解问题, 提出假设, 设计实验
      • 收集数据, 分析数据, 验证假设
    • 应用: 不断基于数据反馈迭代产品 [不断迭代, 更新产品]
      • 制定策略, 实施策略, 反馈迭代
      • 训练算法, 优化算法, 使用算法

观测 :

采集数据 [解析系统日志]
  • 采集方法
    • 传感器采集
    • 爬虫[灰色地带]
    • API
储存数据
  • 各种各样的数据库
    • hive
    • MySQL
    • PostgreSQL
    • SQLServer
    • …
展示数据
  • 可视化高效传达信息

分析数据的目的是什么:

一. 及时发现异常
二. 找到数据之间的因果关系

数据是客观统一的, 有统一的认知, 才能有共同的目标

设定标准 + 发现异常

  • 研究异常数据形成的原因

  • 异常数据也能够拿来使用 [便利贴的发明]

研究关系
  • 可视化查看相关性
  • 建模推导相关性

实验:

提出假设, 然后验证假设
  • 观测到异常 => 提出假设 => 验证假设
所有未经事实数据严振东个想法都是假设
  • 不可以直接根据假设做出决策
设计A/B测试获取数据
  • 具体过程后面会详细介绍
  • 几个问题
    • 业务量只有少数数据时如何设计实验
    • 无法同时测试两个版本时如何比较数据

应用:

如何应用数据创造价值
  • 基于数据反馈不断迭代产品和业务策略
  • 基于数据训练算法, 让机器自动化地完成工作
拆解方法

  • 前提: 明确业务的目标

  • 拆解方法[符合 MECE法则即可, 学会搭建自己的分析框架]

    • 流程拆解法
    • 二分法
    • 象限拆解法
    • 杜邦分析法
    • AARRR
    • PEST
    • RFM
    • SWOT
    • 5W1H

  • 具体流程

    • 明确目标 => 拆解目标 => 观测数据 => 发现异常 => 思考原因 => 拆解问题 => 提出假设
    • 验证假设 => 得到方法 => 制定策略 => 实施策略 => 继续观测 => 明确目标
将数据应用于算法
  • 为算法设定明确的业务目标
  • 为算法提供高质量的数据
  • 判断算法是否真的创造了实际价值
  • 帮助业务更好地使用算法
